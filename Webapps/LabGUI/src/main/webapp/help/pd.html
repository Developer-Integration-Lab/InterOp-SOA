Help - Patient Discovery
  • Response did not return correlation identifiers/ Correlation not found

  • Error/Informative Codes - Patient Discovery


Response did not return correlation identifiers/ Correlation not found:

  • Check the MPI.xml for valid Patient entries on both initiator and responder.
  • Verify the query parameters that are being used in query criteria.
  • Verify the livingSubjectId is present in the request.
  • Verify that the REQUIRED query parameters (LivingSubjectName,LivingSubjectAdministrativeGender,LivingSubjectBirthTime) are present in the request.
  • If the policy engine is turned on, verify the responder Access Consent Policy(ACP) documents.


Response not found:

  • Verify the target gateway "HomecommunityId" in the pateint discovery request.
  • Verify the internalConnectionInfo.xml/uddiConnectionInfo.xml for the correct end points of the initiator and target gateways.
  • Verify that the REQUIRED query parameters (LivingSubjectName,LivingSubjectAdministrativeGender,LivingSubjectBirthTime) are present in the request.
  • Verify the Connectivity to the target server. Check the server logs for connection related errors (for example: connection timed out).
  • Verify the livingSubjectId is present in the request.
  • Verify the certificates are installed correctly and the server was configured with certificates information on initiator and target gateways.
  • If the policy engine is turned on, verify Access Consent Policy(ACP) documents on both initiator and responder ends.
  • Verify logs for any exceptions.


Test Status - FAIL:

  1. Patient correlation record not updated in correlatedIdentifiers table, when correlation expected.
  2. Patient correlation record updated in correlatedIdentifiers table, when correlation not expected.
  3. Check the help topics "Response not found" and "esponse did not return correlation identifiers/Correlation not found" on this page.


Test Status - PROGRESS_PENDING:

  1. Initiator or target servers are not responding.
  2. Test case execution might have caused an exception in the code. Verify the logs.


Case Result (QD) not saved:

  • Verify that the database is up and running at the initiator end.
  • Check the connectivity if the application server and database are on different machines.
  • Data might be too large or attempted to insert null in not null columns. Verify logs for a SQL exception causing this problem.


Error/Informative Codes - Patient Discovery:

  • ResponderBusy: The responder was not able to process the request because it is currently overloaded.
  • AnswerNotAvailable: The answer is not available. Human intervention may be needed.
  • LivingSubjectAdministrativeGenderRequested: Requests the LivingSubjectAdministrativeGender attribute be specified.
  • PatientAdressRequested: Requests the PatientAddress attribute be specified.
  • PatientTelecomRequested: Requests the PatientTelecom attribute be specified.
  • LivingSubjectBirthPlaceNameRequested: Requests the LivingSubjectBirthPlaceName attribute be specified.
  • LivingSubjectBirthPlaceAddressRequested: Requests the LivingSubjectBirthPlaceAddress attribute be specified.
  • MothersMaidenNameRequested: Requests the MothersMaidenName attribute be specified.
