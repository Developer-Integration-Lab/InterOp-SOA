Help - Retrieve Documents
  • Retrieve Document returned zero(0) documents
  • Retrieve documents response not found/ Retrieve Document did not find any documents
  • Test Status - FAIL
  • Test Status - PROGRESS_PENDING
  • Case Result (QD) not saved/ ResultDocument (QD) not saved
  • Error codes - Retrieve Documents


Retrieve Document returned zero(0) documents:

  • Verify the required request parameters (document unique id, repository unique id and home community id) are present and valid
  • Verify that the documents are present in the repository for the given criteria
  • Verify that the target community has the repository configured for the given repository unique id in the document search criteria
  • Invalid query criteria that might have caused an error from the registry in processing the query. Verify logs
  • If the policy engine is turned on, verify the Access Consent Policy(ACP) documents are present
  • Document meta data might be present in the repository, but the actual document might not be there

Retrieve documents response not found/ Retrieve Document did not find any documents:

  • Verify the target gateway "Home Community Id" in the document request
  • Verify that there is atleast one document (document unique id an repository unique id) data is present in the document request
  • Verify the internalConnectionInfo.xml/uddiConnectionInfo.xml for the correct end points of the initiator and target gateways
  • Verify the Connectivity to the target server
  • Verify the certificates are installed correctly and the server was configured with certificates information on initiator and target gateways
  • If the policy engine is turned on, verify Access Consent Policy(ACP) documents on both initiator and responder ends
  • Verify logs for any exceptions

Test Status - FAIL:

  • Returned documents are more or less than expected: Verify the request parameter values in the document request. Verify the documents in repository
  • If expecting single dynamic document, verify that the participant "Dynamic Content" was set to "Single" while registering the participant
  • Check the help topics "Retrieve documents response not found/ Retrieve Document did not find any documents" and "Retrieve Document returned zero(0) documents" on this page

Test Status - PROGRESS_PENDING:

  • Initiator or target servers are not responding
  • Test case execution might have caused an exception in the code. Verify the logs

Case Result (QD) not saved/ ResultDocument (QD) not saved:

  • Verify that the database is up and running at initiator end
  • Check the connectivity if the application server and database are on different machines
  • Data might be too large or attempted to insert null in not null columns. Verify logs

Error codes - Retrieve Documents:

  • XDSRepositoryError: Internal Repository Error
  • XDSRepositoryBusy: Too much activity
  • XDSRepositoryOutOfResources: Resources are low
  • XDSUnknownRepositoryId: The repositoryUniqueId value could not be resolved to a valid document repository or the value does not match the repositoryUniqueId of the Document Repository
  • XDSDocumentUniqueIdError: The document associated with the DocumentUniqueId is not available. This could be because the document is not available to the Document Repository, the requestor is not authorized to access that document or the document is no longer available.
  • XDSUnknownCommunity: A value for the homeCommunityId is not recognized
  • XDSMissingHomeCommunityId: A value for the homeCommunityId is required and has not been specified
